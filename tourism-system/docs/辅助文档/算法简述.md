# 旅游系统算法简述

## 概述

本旅游系统采用了多种经典算法来实现智能推荐、路径规划、内容搜索、数据压缩等功能。本文档详细介绍每个算法的原理、实现方式以及在项目中的具体应用。

---

## 1. 排序算法 (Sorting Algorithms)

### 1.1 快速排序 (Quick Sort)
**算法原理：**
- 分治算法，选择基准元素，将数组分为小于和大于基准的两部分
- 递归对两部分进行排序
- 平均时间复杂度：O(n log n)，最坏情况：O(n²)

**项目中的应用：**
- **场所页面 (PlacesPage)**：对场所进行多键排序，结合热度、评分等多个维度
- **日记页面 (DiariesPage)**：对日记按相关性排序
- **推荐系统**：对推荐结果进行最终排序

**实现位置：** `backend/algorithms/sorting_algorithm.py` - `quick_sort()`

### 1.2 快速选择算法 (Quick Select) - Top-K优化
**算法原理：**
- 基于快速排序的分区思想，只处理包含第K个元素的部分
- 避免完全排序，平均时间复杂度：O(n)
- 自适应策略：小数据集直接排序，大数据集使用快速选择

**项目中的应用：**
- **场所浏览**：按热度/评价排序时只获取前12个结果
- **推荐系统**：所有推荐算法都使用Top-K优化
- **美食搜索**：获取前K个最热门美食

**实现位置：** `backend/algorithms/sorting_algorithm.py` - `quickselect_top_k()`

**性能优化效果：**
- 大规模数据下最高1.86倍加速比
- 内存使用：O(1)空间复杂度
- API响应时间：平均0.018759秒

## 2. 搜索算法 (Search Algorithms)

### 2.1 线性搜索 (Linear Search)
**算法原理：**
- 顺序遍历数据，逐一比较
- 时间复杂度：O(n)
- 适用于无序数据的精确匹配

**项目中的应用：**
- **日记搜索**：按标题精确搜索
- **场所搜索**：按名称精确匹配
- **用户查找**：根据用户ID查找

**实现位置：** `backend/algorithms/search_algorithm.py` - `linear_search()`

### 2.2 二分搜索 (Binary Search)
**算法原理：**
- 在有序数据中查找，每次排除一半数据
- 时间复杂度：O(log n)
- 要求数据预先排序

**项目中的应用：**
- **索引查找**：在构建的搜索索引中快速定位
- **排序后数据查找**：在已排序的结果中快速查找

**实现位置：** `backend/algorithms/search_algorithm.py` - `binary_search()`

### 2.3 模糊搜索 (Fuzzy Search)
**算法原理：**
- 基于Jaccard相似度计算字符串相似性
- 支持容错匹配，处理拼写错误
- 相似度阈值可调节

**项目中的应用：**
- **场所搜索**：支持模糊匹配场所名称
- **设施查找**：容错查找设施名称
- **建筑搜索**：模糊匹配建筑名称

**实现位置：** `backend/algorithms/search_algorithm.py` - `fuzzy_search()`

### 2.4 TF-IDF全文检索算法
**算法原理：**
- **TF (Term Frequency)**：词频 = 词在文档中出现次数 / 文档总词数
- **IDF (Inverse Document Frequency)**：逆文档频率 = log(总文档数 / 包含该词的文档数)
- **TF-IDF分数** = TF × IDF
- 综合考虑词的重要性和稀有性

**项目中的应用：**
- **日记内容搜索**：在旅游日记页面，用户搜索日记内容时使用
- **API调用**：`POST /api/diaries/search?type=content`
- **搜索流程**：
  1. 用户在日记页面输入搜索关键词
  2. 前端调用 `searchDiariesByContent(content)` API
  3. 后端使用TF-IDF算法计算每篇日记与查询词的相关性
  4. 按相关性分数排序返回结果

**实现位置：** `backend/algorithms/search_algorithm.py` - `full_text_search()`

**算法步骤：**
1. **文本预处理**：分词、去除标点符号、转小写
2. **TF计算**：统计查询词在每篇日记中的词频
3. **IDF计算**：计算查询词在整个日记集合中的逆文档频率
4. **相关性评分**：计算每篇日记的TF-IDF总分
5. **结果排序**：按相关性分数降序排列

## 3. 推荐算法 (Recommendation Algorithms)

### 3.1 基于内容的推荐 (Content-Based Filtering)
**算法原理：**
- 分析用户历史偏好，构建用户兴趣档案
- 计算物品特征与用户偏好的相似度
- 使用余弦相似度衡量相似性

**项目中的应用：**
- **场所推荐**：根据用户访问历史推荐相似场所
- **日记推荐**：基于用户兴趣标签推荐相关日记
- **API调用**：`POST /api/places/recommend?algorithm=content`

**实现位置：** `backend/algorithms/recommendation_algorithm.py` - `content_based_recommendation()`

### 3.2 协同过滤推荐 (Collaborative Filtering)
**算法原理：**
- 基于用户行为相似性进行推荐
- 计算用户间的Jaccard相似度
- 综合考虑兴趣、类别偏好、访问历史

**项目中的应用：**
- **个性化推荐**：找到相似用户，推荐他们喜欢的内容
- **冷启动优化**：为新用户提供基于群体行为的推荐

**实现位置：** `backend/algorithms/recommendation_algorithm.py` - `collaborative_filtering_recommendation()`

### 3.3 混合推荐算法 (Hybrid Recommendation)
**算法原理：**
- 结合内容推荐(权重0.6)和协同过滤(权重0.4)
- 综合多种推荐策略的优势
- 提高推荐准确性和多样性

**项目中的应用：**
- **默认推荐策略**：场所页面的个性化推荐
- **API调用**：`POST /api/places/recommend?algorithm=hybrid`

**实现位置：** `backend/algorithms/recommendation_algorithm.py` - `hybrid_recommendation()`

### 3.4 基于热度的推荐 (Popularity-Based)
**算法原理：**
- 综合热度(权重0.6)和评分(权重0.4)
- 用于冷启动场景
- 推荐当前最受欢迎的内容

**项目中的应用：**
- **新用户推荐**：无历史数据时的默认推荐
- **热门内容展示**：首页热门场所推荐

**实现位置：** `backend/algorithms/recommendation_algorithm.py` - `popularity_based_recommendation()`

## 4. 路径规划算法 (Path Planning Algorithms)

### 4.1 Dijkstra最短路径算法
**算法原理：**
- 单源最短路径算法，使用优先队列优化
- 时间复杂度：O((V + E) log V)
- 支持加权图，保证找到最优解

**项目中的应用：**
- **单点路径规划**：计算两个地点间的最短路径
- **多交通工具支持**：步行、自行车、电瓶车
- **实时路况考虑**：结合拥挤度因子调整路径权重
- **API调用**：`POST /api/routes/single`

**实现位置：** `backend/algorithms/shortest_path_algorithm.py` - `dijkstra()`

**路径策略：**
- **最短时间**：考虑交通工具速度和拥挤度
- **最短距离**：仅考虑物理距离
- **混合交通工具**：支持路径中切换交通工具

### 4.2 TSP旅行商问题算法
**算法原理：**
- **最近邻算法**：贪心策略，每次选择最近的未访问节点
- **动态规划算法**：精确解，适用于小规模问题
- 解决多目的地路径优化问题

**项目中的应用：**
- **多目的地路径规划**：优化游览多个景点的路径
- **API调用**：`POST /api/routes/multi`
- **算法选择**：
  - 目的地≤8个：使用动态规划(精确解)
  - 目的地>8个：使用最近邻算法(近似解)

**实现位置：** `backend/algorithms/shortest_path_algorithm.py`
- `tsp_nearest_neighbor()` - 最近邻算法
- `tsp_dynamic_programming()` - 动态规划算法

**特色功能：**

- **路径描述**：生成详细的导航指令
- **楼层支持**：多楼层建筑内导航

## 6. 数据压缩算法 (Data Compression)

### 6.1 Huffman无损压缩算法
**算法原理：**
- 基于字符频率构建最优二叉树
- 高频字符使用短编码，低频字符使用长编码
- 无损压缩，保证数据完整性

**项目中的应用：**
- **日记内容压缩**：压缩存储旅游日记文本内容
- **存储优化**：减少数据库存储空间
- **传输优化**：减少网络传输数据量

**实现位置：** `backend/algorithms/huffman_compression.py`
- `HuffmanCompression` - 核心压缩算法
- `DiaryCompressionManager` - 日记压缩管理器

**压缩流程：**
1. **频率统计**：统计文本中每个字符的出现频率
2. **构建Huffman树**：使用最小堆构建最优编码树
3. **生成编码表**：为每个字符生成唯一的二进制编码
4. **文本编码**：将原文本转换为压缩的二进制串
5. **存储优化**：将二进制串转换为Base64存储

**压缩效果：**
- 平均压缩率：30-50%
- 智能压缩策略：短文本跳过压缩，长文本自动压缩
- 支持批量压缩和解压缩
