# SystemCraft 项目简介

## 1. 项目定位

SystemCraft 是一个面向软件工程教学场景的多 Agent 协作平台。它把一个软件项目从需求分析、架构设计、开发实现、测试评审到复盘总结的全过程串联起来，让用户不仅能看到最终结果，还能看到完整的工程协作过程。

它的核心特点不是“只生成答案”，而是“把软件工程过程可视化、结构化、可复盘化”。

## 2. 项目使用的核心技术

### 后端技术

- Python
- FastAPI
- WebSocket
- SQLite
- LangChain OpenAI 兼容调用
- DeepSeek API
- python-dotenv

后端主要负责项目创建、工作流调度、状态管理、实时消息推送、账号登录和数据持久化。

### 前端技术

- React 18
- TypeScript
- Vite
- Tailwind CSS
- Framer Motion
- React Router
- React Markdown
- ReactFlow

前端主要负责项目创建页面、实时工作台、学习中心、成果展示页和文档导出中心。

### AI 与流程编排

- 多 Agent 分工协作
- 共享状态对象驱动阶段流转
- QA 阶段条件回流机制
- 项目完成后支持 Learning Agent 继续追问与知识沉淀

## 3. 多 Agent 协作流程

SystemCraft 当前采用 5 个主 Agent 完成项目主流程：

1. PM Agent：负责需求分析，输出结构化需求文档。
2. Arch Agent：负责架构设计，输出技术选型、模块划分、数据库和 API 方案。
3. Dev Agent：负责开发实现，输出项目结构、核心代码和配置方案。
4. QA Agent：负责测试评审，检查问题并决定是否回到开发阶段修正。
5. Mentor Agent：负责复盘总结，输出导师讲解和知识卡片。

项目完成后，系统还提供一个补充角色：

- Learning Agent：负责回答学习者的追问，并按需把回答追加为新的知识卡片。

## 4. 协作流程说明

整个流程可以概括为：

`需求输入 -> PM 分析 -> 架构设计 -> 开发实现 -> QA 测试 -> 导师总结`

其中最关键的机制是 QA 回流：

- 如果 QA 发现问题，流程会返回 Dev Agent 进行修正。
- 如果测试通过，流程会进入 Mentor Agent 总结阶段。
- 这样可以更真实地模拟软件团队中的“开发 - 测试 - 修正”闭环。

## 5. 项目主要功能

- 输入一句需求，自动生成完整的软件工程阶段产物
- 在工作台中实时查看 Agent 协作消息和阶段进度
- 查看需求文档、架构方案、代码产物、测试报告和导师总结
- 项目完成后继续向导师提问
- 把追问结果沉淀为知识卡片
- 对已完成项目发起新一轮迭代
- 导出 Markdown 文档和 HTML 展示页

## 6. 项目优势

### 1. 不只给结果，还展示过程

很多 AI 项目只输出一份答案，而 SystemCraft 会把需求、设计、开发、测试和复盘全过程完整呈现出来，更适合教学和项目演示。

### 2. 多角色协作更贴近真实软件团队

项目不是由单一模型一次性完成，而是拆分为 PM、架构师、开发、测试、导师等不同角色，能更真实地体现软件工程的分工思想。

### 3. 支持测试回流，体现工程闭环

QA Agent 不只是“补一句测试结果”，而是能够触发返工流程，使系统具备更真实的工程协作逻辑。

### 4. 学习属性强

Mentor Agent 和 Learning Agent 让系统不仅能“生成项目”，还能帮助用户“理解项目”，非常适合课程作业、课堂展示和自学复盘。

### 5. 产物完整，便于展示与归档

系统生成的内容包含需求、架构、代码、测试、总结和知识卡片，并支持导出，非常适合用于汇报、演示和课程提交。

## 7. 总结

SystemCraft 是一个把“AI 生成能力”和“软件工程教学流程”结合起来的项目。它的价值不只是自动生成软件设计内容，更在于把多 Agent 协作、工程阶段衔接和学习复盘机制整合成一个完整平台。

如果用一句话概括这个项目：

**SystemCraft 是一个用多 Agent 协作方式，演示和教学软件工程全流程的智能项目工作台。**
