## P0 — 急救：让数据链路跑通（30 分钟）

### 目标

让 `spot_xhs_info` 表产生第一条数据，验证整条链路可用。

### Step 1：获取有效的小红书 Cookie

```
1. 打开 Chrome，访问 https://www.xiaohongshu.com
2. 扫码或手机号登录你的小红书账号
3. 登录成功后，按 F12 → Application → Cookies → www.xiaohongshu.com
4. 复制以下关键 Cookie 字段（至少要包含 a1 和 web_session）：
   - a1=...
   - web_session=...
   - webId=...
5. 拼接为一行字符串（用分号+空格分隔）：
   a1=18...; web_session=...; webId=...
```

### Step 2：写入 Agent 环境变量

编辑 `agent-service/.env`：

```bash
# 粘贴刚才复制的 Cookie 字符串
XHS_SYSTEM_COOKIES_STR=a1=18...; web_session=...; webId=...

# 生成加密密钥（如果还没有）
XHS_COOKIE_ENCRYPTION_KEY=<用 python -c "import secrets; print(secrets.token_hex(32))" 生成>
```

### Step 3：重启 Agent 服务并验证

```powershell
# 重启 Agent
cd agent-service
.\.venv\Scripts\python.exe -c "
import os
os.environ['XHS_SYSTEM_COOKIES_STR'] = '你的cookie'
from app.skills.xhs_client import XHSClient
c = XHSClient()
r = c.search_notes('西湖', num=3)
print('ok:', r.get('ok'), 'total:', r.get('data',{}).get('total',0))
"
```

预期输出：`ok: True total: <大于0的数字>`

### Step 4：手动触发一次 refresh

```powershell
# 调用 Agent 刷新端点
curl -X POST "http://localhost:9000/agent/xhs/refresh-place?place_name=西湖&place_id=<你数据库中某个景点的id>&place_type=scenic"
```

或通过 Java 后端：

```powershell
curl -X POST "http://localhost:8080/api/places/<place_id>/refresh-xhs" -H "Authorization: Bearer <token>"
```

### Step 5：验证首页

刷新前端首页，检查"发现小红书热门打卡地"板块是否从跳转按钮变为卡片网格。

### 验证清单

| 检查点 | 命令/方法 | 预期结果 |
|--------|----------|----------|
| Cookie 有效 | Agent `/agent/xhs/search?query=西湖&num=3` | 返回笔记列表 |
| 抓取成功 | `POST /agent/xhs/refresh-place?place_name=西湖&place_id=xxx` | `{"ok": true, "trending_score": >0}` |

| 首页展示 | 打开前端首页 | 显示 4 张热门卡片 |

## P1 — 治本：批量种子脚本 + 启动自检（2 小时）

### 目标

系统启动时自动为所有景点填充 XHS 数据，消灭"鸡生蛋"死锁。

### 方案一：编写独立种子脚本（推荐）

新建文件 `agent-service/scripts/seed_xhs_data.py`：

```python
"""
XHS 数据种子脚本 — 为系统中所有景点批量抓取小红书热度数据。
每次部署/重启时运行一次即可。

用法:
    python scripts/seed_xhs_data.py [--force] [--delay 3]
"""
import json
import os
import sys
import time
import urllib.request
import urllib.error

BACKEND = os.getenv("TOURISM_BACKEND_BASE_URL", "http://127.0.0.1:8080")
AGENT = os.getenv("AGENT_BASE_URL", "http://127.0.0.1:9000")
DELAY = float(os.getenv("SEED_DELAY", "3"))  # 每个景点之间间隔秒数，避免触发反爬
FORCE = "--force" in sys.argv


def get_all_places():
    """从 Java 后端获取所有景点列表."""
    url = f"{BACKEND}/api/places"
    req = urllib.request.Request(url)
    try:
        with urllib.request.urlopen(req, timeout=10) as resp:
            data = json.loads(resp.read())
            if data.get("code") == 200 and data.get("data"):
                return data["data"]
    except urllib.error.URLError as e:
        print(f"[ERROR] 无法连接后端: {e}")
    return []


def get_xhs_info(place_id):
    """检查景点是否已有 XHS 数据."""
    url = f"{BACKEND}/api/places/{place_id}"
    req = urllib.request.Request(url)
    try:
        with urllib.request.urlopen(req, timeout=5) as resp:
            data = json.loads(resp.read())
            detail = data.get("data", {})
            xhs = detail.get("xhsInfo")
            if xhs and xhs.get("trendingScore", 0) > 0:
                return xhs
    except Exception:
        pass
    return None


def refresh_place(place_name, place_id, place_type="general"):
    """调用 Agent 刷新单个景点的 XHS 数据."""
    url = (
        f"{AGENT}/agent/xhs/refresh-place"
        f"?place_name={urllib.parse.quote(place_name)}"
        f"&place_id={urllib.parse.quote(place_id)}"
        f"&place_type={urllib.parse.quote(place_type)}"
    )
    req = urllib.request.Request(url, method="POST")
    try:
        with urllib.request.urlopen(req, timeout=60) as resp:
            return json.loads(resp.read())
    except urllib.error.URLError as e:
        return {"ok": False, "error": str(e)}


def main():
    print("=" * 60)
    print("XHS 数据种子脚本")
    print("=" * 60)

    # 1. 获取所有景点
    places = get_all_places()
    if not places:
        print("[FAIL] 没有获取到任何景点，请检查后端是否启动")
        sys.exit(1)
    print(f"[INFO] 共获取到 {len(places)} 个景点")

    # 2. 过滤出需要刷新的景点（无数据 或 force）
    todo = []
    for p in places:
        pid = p.get("id")
        pname = p.get("name")
        if not pid or not pname:
            continue
        if FORCE:
            todo.append(p)
        else:
            xhs = get_xhs_info(pid)
            if xhs is None:
                todo.append(p)
                print(f"  [MISS] {pname} ({pid}) — 无 XHS 数据，加入队列")
            else:
                print(f"  [SKIP] {pname} ({pid}) — 已有数据, trending={xhs.get('trendingScore')}")

    if not todo:
        print("[DONE] 所有景点已有 XHS 数据，无需刷新")
        return

    print(f"\n[INFO] 将刷新 {len(todo)} 个景点 (间隔 {DELAY}s)\n")

    # 3. 逐个刷新
    ok_count = 0
    fail_count = 0
    for i, p in enumerate(todo):
        pid = p.get("id")
        pname = p.get("name")
        ptype = p.get("type", "general")

        print(f"[{i+1}/{len(todo)}] 刷新: {pname} ... ", end="", flush=True)
        result = refresh_place(pname, pid, ptype)

        if result.get("ok"):
            score = result.get("trending_score", 0)
            count = result.get("note_count", 0)
            print(f"OK (热度={score}, 笔记={count})")
            ok_count += 1
        else:
            error = result.get("error", "unknown")
            print(f"FAIL: {error}")
            fail_count += 1

        # 间隔等待，避免触发反爬
        if i < len(todo) - 1:
            time.sleep(DELAY)

    # 4. 汇总
    print(f"\n{'=' * 60}")
    print(f"完成: 成功 {ok_count}, 失败 {fail_count}, 总计 {len(todo)}")
    print(f"{'=' * 60}")

    if fail_count > 0:
        print("[WARN] 部分景点刷新失败，可能是 Cookie 过期或网络问题")
        print("[WARN] 待 Cookie 更新后重新运行此脚本即可")


if __name__ == "__main__":
    main()
```

### 方案二：Agent 启动时自动预热（备选）

在 `agent-service/app/main.py` 的 `startup_event` 中增加：

```python
@app.on_event("startup")
async def startup_event():
    # ... 现有启动逻辑 ...

    # 异步预热 XHS 数据（不阻塞启动）
    import asyncio
    asyncio.create_task(_warmup_xhs_data())

async def _warmup_xhs_data():
    """后台预热：为没有 XHS 数据的景点批量抓取."""
    import os
    if not os.getenv("XHS_SYSTEM_COOKIES_STR"):
        logger.warning("XHS cookie not configured, skip warmup")
        return

    try:
        # 等 5 秒，确保服务完全启动
        await asyncio.sleep(5)

        from app.tools.xhs_scraper import refresh_place_xhs_data
        import httpx

        async with httpx.AsyncClient() as client:
            resp = await client.get(
                f"{os.getenv('TOURISM_BACKEND_BASE_URL')}/api/places",
                timeout=10
            )
            places = resp.json().get("data", [])

        warmup_count = 0
        for p in places:
            pid = p.get("id")
            # 检查是否已有数据
            resp = await client.get(
                f"{os.getenv('TOURISM_BACKEND_BASE_URL')}/api/places/{pid}",
                timeout=5
            )
            detail = resp.json().get("data", {})
            if detail.get("xhsInfo", {}).get("trendingScore", 0) > 0:
                continue

            # 无数据 → 抓取
            result = refresh_place_xhs_data(
                place_name=p.get("name"),
                place_id=pid,
                place_type=p.get("type", "general"),
            )
            if result.get("ok"):
                # 写入后端数据库
                await client.post(
                    f"{os.getenv('TOURISM_BACKEND_BASE_URL')}/api/places/{pid}/refresh-xhs",
                    json=result,
                    timeout=30,
                )
                warmup_count += 1
                logger.info(f"XHS warmup: {p.get('name')} OK")

            # 间隔 3 秒
            await asyncio.sleep(3)

        logger.info(f"XHS warmup complete: {warmup_count} places refreshed")

    except Exception as e:
        logger.warning(f"XHS warmup failed (non-fatal): {e}")
```

**选哪个？** 推荐**方案一（独立脚本）**。理由：
- 脚本可独立运行、重跑、调试，不绑死 Agent 生命周期
- Agent 启动时异步预热会影响启动速度，且难以观测进度
- 脚本可以在 CI/CD、Docker entrypoint 中灵活调用

### Cookie 健康检查（配套措施）

Agent 启动时增加 Cookie 有效性检查。在 `agent-service/app/main.py` startup 中：

```python
# 启动时验证 XHS Cookie 是否有效
if os.getenv("XHS_SYSTEM_COOKIES_STR"):
    try:
        from app.skills.xhs_client import XHSClient
        client = XHSClient()
        result = client.validate_cookie(os.getenv("XHS_SYSTEM_COOKIES_STR"))
        if result.get("ok"):
            logger.info(f"XHS Cookie valid — user: {result.get('xhs_username')}")
        else:
            logger.warning(
                f"XHS Cookie INVALID: {result.get('error')}. "
                f"首页热门打卡地将显示为空。请更新 .env 中的 XHS_SYSTEM_COOKIES_STR"
            )
    except Exception as e:
        logger.warning(f"XHS Cookie check skipped: {e}")
else:
    logger.warning(
        "XHS_SYSTEM_COOKIES_STR not set. "
        "小红书热门打卡地功能将不可用。"
    )
```

这样运维人员启动服务时能立刻看到问题，而非等用户反馈。

## P2 — 兜底：首页降级策略优化（1 小时）

### 问题

当前兜底态的按钮文案"探索热门地点"给用户错误预期——点击后只是跳转到普通的目的地搜索页，并没有"探索热门"的体验。

### 方案：三态降级

修改 `HomePage.jsx` 的兜底逻辑，不再用单一的 fallback banner，而是根据数据缺失原因展示不同的状态：

```jsx
// HomePage.jsx — 替换 lines 137-157 的 fallback banner

{trendingPlaces.length === 0 && (
  <section className="mb-12">
    <div className="border border-rose-200 bg-gradient-to-r from-rose-50 to-white rounded-2xl p-6">
      <div className="flex items-center gap-3 mb-4">
        <span className="text-2xl">📕</span>
        <div>
          <h3 className="font-serif text-base text-heading">发现小红书热门打卡地</h3>
          <p className="text-xs text-muted mt-0.5">基于小红书用户真实笔记的热度排行</p>
        </div>
      </div>

      {/* 降级态 A：系统自有高评分景点（无需 XHS 数据） */}
      {topRatedPlaces.length > 0 ? (
        <>
          <p className="text-xs text-muted mb-4">
            ⚠️ 小红书热度数据正在准备中，以下为系统内高评分推荐
          </p>
          <div className="grid grid-cols-2 md:grid-cols-4 gap-3">
            {topRatedPlaces.slice(0, 4).map(place => (
              <div
                key={place.id}
                onClick={() => navigate(`/places/${place.id}`)}
                className="group cursor-pointer p-3 rounded-xl hover:bg-rose-50/60 transition-colors"
              >
                <div className="aspect-square bg-accent-soft rounded-xl overflow-hidden mb-2">
                  {place.image ? (
                    <img src={getFileUrl(place.image)} alt={place.name}
                      className="w-full h-full object-cover group-hover:scale-105 transition-transform duration-500" />
                  ) : (
                    <MapPin size={24} className="w-full h-full p-6 text-muted/40" />
                  )}
                </div>
                <p className="font-serif text-sm text-heading truncate">{place.name}</p>
                <StarRating value={place.rating || 0} size={10} className="mt-1" />
              </div>
            ))}
          </div>
        </>
      ) : (
        /* 降级态 B：连系统数据都没有 */
        <div className="text-center py-4">
          <p className="text-sm text-muted mb-2">热门景点数据正在准备中</p>
          <p className="text-xs text-muted/60">请稍后再来探索</p>
        </div>
      )}
    </div>
  </section>
)}
```

同时修改数据获取逻辑，在 `trendingPlaces` 为空时准备 fallback：

```jsx
// 在 useEffect 中增加 fallback 逻辑
useEffect(() => {
  getPlaces().then(res => {
    if (res.success && res.data) {
      // 有 XHS 数据的 → trendingPlaces
      const withXhs = res.data.filter(p => p.xhsInfo?.trendingScore > 0);
      const sorted = [...withXhs]
        .sort((a, b) => (b.xhsInfo?.trendingScore || 0) - (a.xhsInfo?.trendingScore || 0))
        .slice(0, 4);
      setTrendingPlaces(sorted);

      // 无 XHS 但评分高的 → 降级展示用
      const withoutXhs = res.data
        .filter(p => !p.xhsInfo || p.xhsInfo.trendingScore === 0)
        .sort((a, b) => (b.rating || 0) - (a.rating || 0));
      setTopRatedFallback(withoutXhs);
    }
  }).catch(() => {});
}, []);
```

## 费用与风险

| 方案 | 风险 | 缓解 |
|------|------|------|
| P0 手动配 Cookie | Cookie 会在几天后过期 | P1 的健康检查会在启动时告警 |
| P1 种子脚本 | 批量请求触发小红书反爬 | 每次请求间隔 3 秒（`SEED_DELAY`），按需运行不自动循环 |
| P1 种子脚本 | 景点数量多时耗时较长 | 仅处理无数据景点（增量），且可指定 `--force` 控制 |
| P2 降级展示 | 无 XHS 数据时展示系统数据，可能让用户觉得"不够小红书" | 明确标注"数据准备中"，管理预期 |
