# 跳跃性创新提案

> 目标：脱离"行程规划/推荐系统/语音导览"的老套叙事，找一些年轻人会主动截图发群里的东西。

---

## 一、扔骰子旅行 — 随机冒险生成器

**不是**推荐系统告诉你"评分最高的景点"。

**而是**你摇一摇手机，系统给你一个随机任务：

> 🎲 掷出 3 点  
> 📍 任务：找到离你最近的一棵树，拍下树影  
> 🍜 然后去吃一份你没吃过的口味的冰淇淋  
> 📝 用三个emoji记录此刻心情并写一句话  
> ⏱️ 限时 30 分钟完成，超时请发朋友圈说"我输了"

每次掷骰子生成 1 个随机"微任务"——可能是去某个地方、拍某类照片、吃某种食物、跟陌生人说一句话、写一句话。任务不是凭空生成的，而是基于你当前位置周边真实存在的场所和美食来拼接。完成有积分，失败有惩罚。

**年轻在哪**：它不是帮你"高效旅行"，而是逼你"无意义冒险"。像 BeReal 一样制造意外，像 Wordle 一样有每日挑战。

**现有数据支撑**：场所坐标 + 美食数据 + 前端定位 → 随机约束生成，80% 可复用。

**传播钩子**：任务结果一键生成分享卡片——"今天命运的安排是..."，好友可以点进来接受同一个任务挑战。

## 二、旅行发疯文学生成器

**不是**DiaryAgent 帮你写一篇正经游记。

**而是**你把照片丢进去，它用当代互联网语体帮你写：

> 输入：3 张西湖照片  
> 输出（风格选择：发疯文学 / 废话文学 / 咯噔文学 / 舔狗文学）：

**发疯文学**：
> "西湖的水我的泪～～谁懂啊家人们这个波光粼粼真的绝绝子😭💅 我在湖边站了五分钟感觉整个人都被治愈了谢谢大自然谢谢你西湖你是我永远的快乐老家！！！#精神状态良好 #今天也在努力活着"

**废话文学**：
> "这片湖，怎么说呢，就是一片湖。它的特点是：有水。水很多。多到什么程度呢，就是一眼望不到边。望不到边的意思就是，它很大。很大是什么概念呢，就是你站在这里，看不到对面。"

**咯噔文学**：
> "他站在断桥边，风吹起他的衣角。他忽然明白了——不是所有的相遇都有结局，就像不是所有的人在西湖边都会想起白娘子。但他还是想起了。这大概就是，宿命吧。"

做完之后一键复制/生成图片发小红书或朋友圈。

**年轻在哪**：它嘲笑了"旅行必须写有深度的文字"这件事。AI 帮你认真胡说八道。

**现有数据支撑**：DiaryAgent 的 LLM 流水线已通，只需替换 prompt 模板。HTML→Canvas 生成分享卡片用 `html2canvas`，半天搞定。

## 三、赛博算命 — 用迷信包装推荐

**不是**在搜索结果里按评分排序。

**而是**你打开一个黄历风格的界面：

> 🔮 今日运势 · 乙巳年三月十四  
>  
> **宜**：户外活动 · 碳水摄入 · 拍照发朋友圈  
> **忌**：宅在宿舍 · 吃沙拉 · 开省电模式  
>  
> **幸运方向**：西南（步行约 12 分钟）  
> **幸运食物**：麻辣香锅  
> **幸运地点**：第三食堂二楼 · 靠窗第三个座位  
> **今日锦鲤**：下午 4:27 出门会遇到好事  
>  
> *——本运势由算法根据你的历史行为、当前位置和今日天气生成，不准不要钱，准了记得来还愿*

每次刷新得到不同的"运势"，但背后其实是推荐算法——宜忌是根据你的偏好反推的（你平时爱吃辣的 → 今日宜辣），幸运方向是随机选了一个附近评分高的地点。用户分享的是"哈哈这个算命的好准"而不是"这个 app 给我推了个推荐"。

**年轻在哪**：年轻人不信算命但爱算命。用迷信的外壳包装推荐引擎，推荐变得好玩而不是好用。

**现有数据支撑**：推荐算法 + 用户行为数据 + 前端定位，只需换 UI 皮。

**传播钩子**：每日运势卡片可分享，朋友扫二维码进入同一个 app 看自己的运势。

## 四、NPC 人格切换 — 让 AI 助手变成"那个人"

**不是**一个中性的"个性化旅游助手"。

**而是**你把助手切换成不同人格，它用那个角色的语气跟你说话：

| NPC 人格 | 设定 | 对话风格 |
|----------|------|----------|
| **毒舌导游** | 退休老导游，看不惯一切网红打卡 | "又去断桥？那地方除了人头你还能看到什么？听我的，去茅家埠，人少水清，别跟那帮游客挤。" |
| **文艺流浪猫** | 一只自称在西湖边活了三百年的猫 | "喵。你脚下这条路，乾隆走过，苏轼走过，现在轮到你走了。别走太快，左边第三棵柳树上有个鸟窝，替我看看还在不在。" |
| **特种兵教官** | 军事化旅行指挥官 | "0715 到达指定地点！0716 拍照！拍照动作要快！0717 转移！下一目标第三食堂，跑步——走！" |
| **失意诗人** | 什么都想赋诗一首但水平很差的诗人 | "啊！食堂！你是饥饿的终点，是卡路里的起点，是饭卡余额的安息之地..." |
| **暗恋你的学妹** | 全程用校园暗恋视角说话 | "学长/学姐今天去图书馆了吗...啊那个位置！那个位置是我最喜欢的！阳光会在下午三点照到桌角..." |

**实现方式**：本质是 system prompt 切换 + 前端角色选择 UI。完全不改后端逻辑，只改提示词。聊天界面增加一个角色头像和名称，切换角色时清空上下文。

**年轻在哪**：ChatGPT 只有一个性格。但有性格的 AI 才有人愿意跟它玩。角色扮演 + AI = 无限可能。

**现有数据支撑**：Agent 的 `prompts.py` 集中管理提示词，新增 5 组 system prompt 即可。零后端改动。

## 五、电子榨菜 — 自动生成的旅行碎片

**不是**写一篇完整的旅行日记。

**而是**系统从你的浏览轨迹、打卡记录、照片上传中，自动切成一条一条的"旅行碎片"——像 Instagram Stories 或小红书的单张卡片：

> 🕑 14:32 你在图书馆三楼待了 47 分钟  
> 🕒 15:19 你走到了操场，停留 3 分钟，然后走了  
> 🕓 16:05 你在食堂点了一份麻辣香锅（这是你本周第三次吃辣）  
> 🕔 16:52 你拍了一张夕阳  
>  
> **今日旅行碎片已生成**（3 张卡片，左右滑动查看）

每张卡片包含：时间戳 + 一句话描述 + 自动配图（如果有拍照）+ 一个 emoji 情绪标签。用户不需要"写"日记，旅行碎片自动生成，只需选择要不要发布（或删掉不想让别人看到的）。

**年轻在哪**：BeReal + Locket Widget + 小红书 plog 的混合体。年轻人不写日记，但愿意发碎片。它不是工具，是内容消费品。

**现有数据支撑**：用户行为数据（浏览/评分时间戳）+ 图片上传 + DiaryAgent 摘要能力。新增一个"碎片化时间线"视图即可。

## 六、陌生人路线交换 — 旅行盲盒

**不是**你用算法规划最优路线。

**而是**你把今天走过的路线匿名扔进一个池子里，系统随机还你一条某个陌生人今天走过的路线。

> 📤 你分享了今天的路线：3.2 公里，4 个停留点，关键词「闲逛+咖啡+拍照」  
> 📥 你收到了一个陌生人的路线：2.1 公里，3 个停留点，关键词「找吃的+迷路+发现一个小店」  
>  
> 要不要走一遍试试？  
> ⚠️ 你知道的越少，体验越有趣。系统不会告诉你这个人是谁，也不会告诉你路上会遇到什么。

路线的起点终点模糊化处理（只显示距离和大概方向），用户选择"接受挑战"后才显示完整路线。走完之后可以选择"回复"一条消息给路线的主人（也是匿名的），形成一种异步的、轻量级的陌生人社交。

**年轻在哪**：它不是社交网络，没有好友、没有关注、没有点赞。它更像漂流瓶或 Slowly——慢、匿名、有期待感。

**现有数据支撑**：路线保存 → 路线分享接口（新增一个 `POST /api/routes/share` 存入匿名池，`POST /api/routes/blind-box` 随机取出一条）。数据结构简单，核心体验在前端。

## 七、旅行能量条 — 放弃打分，改用精神状态

**不是**给景点打 1-5 星评分。

**而是**用更年轻的维度记录旅行体验：

| 维度 | 选项 |
|------|------|
| 🪫 能量消耗 | 轻松散步 · 正常逛 · 腿已废 |
| 💸 钱包感受 | 白嫖 · 正常 · 破产了 |
| 😤 值不值得 | 血赚 · 还行 · 还我妈生钱 |
| 📸 出片率 | 张张大片 · 勉强能发 · 算了不发了 |
| 🍜 好吃程度 | 舔盘子 · 正常 · 不如泡面 |
| 🫣 社死风险 | 独自美丽 · 人多但可控 · 社恐地狱 |

用户在结束一次行程后，不是填评分表单，而是拉一组滑块——像 iOS 的健康 app 那样，用视觉化的能量条完成"旅行复盘"。系统汇总所有人的评价后在景点详情页展示一个"精神状态雷达图"而不是干巴巴的 4.2 分。

**年轻在哪**：1-5 星是上一代的评价体系。"出片率""社死风险""还我妈生钱"才是这代人的语言。

**现有数据支撑**：`user_behavior` 表增加 `behavior_metadata` JSON 字段存储新维度，不影响原有评分逻辑。

## 八、"别去了"功能 — 反向推荐

**不是**推荐你该去哪。

**而是**告诉你哪些地方**现在不该去**：

> 🚫 **断桥** — 当前人数：拥挤。去了只能看后脑勺。  
> 🚫 **图书馆** — 考试周，一座难求。别去了真的。  
> 🚫 **第三食堂** — 麻辣香锅窗口今日休息。别白跑。  
>  
> ✅ 替代方案：茅家埠（人少）、咖啡馆自习区（有位）、第二食堂（正常营业）

反推荐是另一种推荐。年轻人对"劝退"的信任度远高于"种草"。

**实现方式**：在场所数据中增加实时状态标签（拥挤/关闭/休息），前端展示"劝退卡片"。不需要传感器数据，用规则 + 用户反馈即可：如果一个场所在过去 30 分钟内被 3 个人标记为"人多"，则触发劝退。

## 功能组合：一个"年轻人会分享"的体验闭环

```
上午 10:30
用户打开 app → 首页是赛博算命："今日宜碳水摄入，幸运方向西南"

上午 11:00
用户切换到"毒舌导游" NPC 人格：
"又想去食堂？行吧，第三食堂麻辣香锅，别点微辣，丢人。"
用户被逗笑，截图发群里："哈哈哈哈这个AI骂我"

下午 14:00
用户掷了一次骰子 → 任务是"找到校园里最丑的建筑并合影"
用户在群里直播执行任务

晚上 20:00
系统自动生成今日"电子榨菜"碎片：
"今天你走了 4.2km, 被AI骂了 3 次, 和丑建筑合了影, 麻辣香锅吃了中辣"
用户一键分享到小红书

晚上 20:30
用户觉得今天玩得很开心，把自己的路线扔进盲盒池
换回一条陌生人路线："2.1km, 关键词：找猫+失败+但发现了一个秋千"
用户明天准备去走
```

## 优先级与可行性的诚实评估

| 功能 | 惊艳度 | 实现难度 | 后端改动 | 前端改动 | 建议 |
|------|--------|----------|----------|----------|------|
| NPC 人格切换 | ★★★★★ | ★☆☆☆☆ | 只改 prompt | 加角色选择 UI | **本周可上** |
| 旅行发疯文学 | ★★★★ | ★★☆☆☆ | 替换 DiaryAgent prompt | 加风格选择 + 分享卡片 | **本周可上** |
| 赛博算命 | ★★★★★ | ★★☆☆☆ | 推荐逻辑包装 | 黄历风格 UI | **两周可上** |
| 扔骰子旅行 | ★★★★★ | ★★★☆☆ | 随机任务生成器 | 掷骰子动效 + 任务卡片 | 两周 |
| 电子榨菜 | ★★★★ | ★★★☆☆ | 行为聚合 + 碎片生成 | 时间线 + Stories 风格 | 两周 |
| 旅行能量条 | ★★★★ | ★★☆☆☆ | behavior 表加字段 | 滑块交互 + 雷达图 | 一周 |
| 陌生人路线交换 | ★★★★★ | ★★★☆☆ | 路由分享/盲盒池 | 交换动效 + 匿名聊天 | 三周 |
| 别去了 | ★★★ | ★★☆☆☆ | 状态标签 + 劝退规则 | 劝退卡片 | 一周 |

**建议上手顺序**：NPC 人格 + 发疯文学（改动最小，效果最炸）→ 赛博算命 → 扔骰子旅行 → 电子榨菜 → 陌生人路线交换。
